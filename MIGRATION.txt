# @vrsim/earth-engine-ol 1.x 升级到 2.0.0

2.0.0 是一次完整的主版本重构，不提供 1.x API 兼容层。1.x 的地图、图层、图形、绘制、编辑、测量、Transform、事件、右键菜单、Overlay、Descriptor 和动画能力大多已迁移到新的统一服务中；Wind 已明确删除，普通 Polygon 的多环和洞结构目前也没有 V2 对应能力，具体差异见后文。

建议先在独立分支完成迁移，再升级业务项目中的依赖。

## 1. 安装、ESM 和样式

### 1.1 只提供 ESM

2.0.0 不再提供 CommonJS 入口，不能继续使用 `require()`：

```ts
// 1.x
const { Earth, PointLayer } = require('@vrsim/earth-engine-ol');

// 2.0.0
import { Earth } from '@vrsim/earth-engine-ol';
```

ESM 不需要额外安装插件。Node 直接运行时，请使用 `.mjs` 文件，或在业务项目的 `package.json` 中设置：

```json
{
  "type": "module"
}
```

通过 Vite、Rollup、Webpack 等工具构建的浏览器项目，按构建工具的 ESM 配置使用即可。

### 1.2 只从包根入口导入

2.0.0 只开放包根入口和样式入口：

```ts
import { Earth, useEarth } from '@vrsim/earth-engine-ol';
import '@vrsim/earth-engine-ol/style.css';
```

1.x 正式开放的 `dist/*` 深路径已经删除：

- `@vrsim/earth-engine-ol/dist/*`

以下路径并不是 1.x `package.json` 正式声明的子路径；如果旧项目曾通过打包器或文件路径绕过 exports 深导入，2.0.0 同样不能继续使用：

- `@vrsim/earth-engine-ol/core`
- `@vrsim/earth-engine-ol/layers`
- `@vrsim/earth-engine-ol/draw`
- `@vrsim/earth-engine-ol/measure`
- `@vrsim/earth-engine-ol/transform`
- `@vrsim/earth-engine-ol/plot`

旧样式导入也必须替换：

```ts
// 1.x
import '@vrsim/earth-engine-ol/dist/index.es.css';

// 2.0.0
import '@vrsim/earth-engine-ol/style.css';
```

公开的 `style.css` 已包含 OpenLayers 基础样式和本库组件样式，不需要再导入本包的其他 CSS 文件。

## 2. OpenLayers 10、依赖和离线安装

### 2.1 OpenLayers 由业务项目安装

1.x 使用 `ol@^7.5.2`。2.0.0 的公开基线为 `ol@^10.9.0`，并将 OL 声明为安装时可选的 peer dependency。

```bash
npm install @vrsim/earth-engine-ol@2.0.0 ol@10.9.0
```

`optional` 只表示引擎 tgz 可以单独完成安装，不表示 OL 是可选的地图运行依赖。业务项目没有安装 OL 时，地图功能无法正常构建或运行。

2.0.0 已按 OL 10 调整为显式公开 ESM 模块，不再依赖 OL barrel、私有字段或 renderer 内部类型。业务代码如果直接使用 OL，也应优先从带 `.js` 后缀的公开模块导入：

```ts
import VectorLayer from 'ol/layer/Vector.js';
import VectorSource from 'ol/source/Vector.js';
```

### 2.2 删除普通运行依赖

2.0.0 没有普通、可选或打包运行依赖，并删除了：

- `heatmap.js`
- `lodash`
- `mitt`
- `ol-wind`
- `wind-core`
- `@types/heatmap.js`
- `@types/lodash`

如果业务代码自己使用这些包，应由业务项目显式安装，不要依赖本库的传递安装。

### 2.3 Wind 已删除

`WindLayer`、`WindLayerInstance`、`ol-wind` 和 `wind-core` 已全部删除，没有 V2 替代 API。使用风场的业务必须先移除相关调用，或在业务层自行接入其他风场实现。

### 2.4 离线安装

引擎 tgz 支持在全新空缓存中强制离线安装，安装结果只包含 `@vrsim/earth-engine-ol`。完整业务环境仍需提前准备 OL 10.9.0 及其依赖闭包。

推荐流程：

1. 从正式发布物取得引擎 tgz；仓库维护者也可以在源码仓库执行 `npm pack`。
2. 在有网络的真实业务项目或其 `package.json`、`package-lock.json` 完整副本中，把引擎 tgz 与 `ol@10.9.0` 写入同一锁文件。
3. 按该锁文件执行 `npm ci --cache ./npm-cache`，预热业务项目的完整依赖闭包，而不只是 OL 的闭包。
4. 保持相对路径，将业务依赖文件、引擎 tgz 和已预热的 npm 缓存一起带入离线环境。
5. 使用同一锁文件和缓存执行 `npm ci --offline`，再完成类型检查、构建和运行验证。

引擎包本身可以这样验证：

```bash
npm install --offline --ignore-scripts ./vrsim-earth-engine-ol-2.0.0.tgz
```

完整业务项目应在联网环境生成并验证锁文件，然后在离线环境按同一锁文件安装：

```bash
npm ci --offline --cache ./npm-cache --ignore-scripts --no-audit --no-fund
```

只携带两个裸 tgz 或只预热 OL 依赖都不足以保证完整业务项目可离线安装；业务锁文件中的全部依赖都必须已经存在于缓存或离线制品中。

### 2.5 Node 运行环境

2.0.0 是纯 ESM 包，发布代码目标为 ES2022。`package.json` 已声明并验证以下工具链范围：

- Node `>=24.18.0 <25`
- npm `>=11 <12`

Node 16、18、20、22 和 25 均不在 2.0.0 的发布契约内。即使代码最终只在浏览器运行，安装、类型检查和构建阶段也必须使用上述 Node 与 npm 范围。

## 3. Earth 和 useEarth

### 3.1 构造参数改为单个对象

1.x 的公开签名是 `new Earth(viewOptions?, options?)` 和 `useEarth(viewOptions?, options?)`。2.0.0 必须改为单个 `EarthOptions` 或 `UseEarthOptions` 配置对象：

```ts
// 1.x
const earth = new Earth(
  { center, zoom: 6 },
  { target: 'map', zoom: true, rotate: false }
);

const defaultEarth = useEarth(
  { center, zoom: 6 },
  { target: 'map', zoom: true }
);

// 2.0.0
const earth = new Earth({
  target: 'map',
  view: { center, zoom: 6 },
  controls: { zoom: true, rotate: false }
});

const defaultEarth = useEarth({
  target: 'map',
  view: { center, zoom: 6 },
  controls: { zoom: true }
});
```

`target` 支持非空字符串或 `HTMLElement`。未知字段、访问器属性和非法值会直接报错。

### 3.2 useEarth 没有就创建，有就返回

2.0.0 固定提供三个重载：

```ts
useEarth(): Earth;
useEarth(id: string): Earth;
useEarth(options: UseEarthOptions): Earth;
```

默认实例：

```ts
const earth = useEarth();
console.assert(useEarth() === earth);
```

命名实例：

```ts
const planning = useEarth('planning');
const overview = useEarth('overview');

console.assert(useEarth('planning') === planning);
console.assert(useEarth('overview') === overview);
console.assert(planning !== overview);
```

带配置的命名实例：

```ts
const planning = useEarth({
  id: 'planning',
  target: 'planning-map',
  view: { center, zoom: 7 },
  controls: { attribution: false }
});
```

规则如下：

- `useEarth()` 使用默认注册键，首次默认挂载到 `olContainer`。
- `useEarth(id)` 使用命名注册键，首次省略 target 时用 id 作为 target。
- `useEarth(options)` 通过可选 id 选择默认或命名注册键。
- 同一注册键的 `target`、`view` 和 `controls` 仅在首次创建时生效。
- 实例仍活动时，后续调用始终返回原对象，不会用新配置隐式重建。
- 开发环境遇到明显的冲突配置时会给出 warning。
- `new Earth(options)` 创建调用方自管实例，不会进入 `useEarth` 注册表。

### 3.3 destroyEarth 已删除

1.x：

```ts
import { destroyEarth } from '@vrsim/earth-engine-ol';

destroyEarth();
```

2.0.0 必须保留实例引用并直接销毁：

```ts
const earth = useEarth();
earth.destroy();

const planning = useEarth('planning');
planning.destroy();
```

`destroyEarth()`、`destroyEarth(id)`、`createEarth()` 和 `getEarth()` 都不存在。

`earth.destroy()` 是幂等操作。注册实例销毁后会自动注销；之后使用同一注册键调用 `useEarth` 会创建新实例：

```ts
const previous = useEarth('planning');
previous.destroy();

const current = useEarth('planning');
console.assert(current !== previous);
```

生命周期可以通过以下属性读取：

```ts
earth.lifecycle;  // 'ready' | 'destroying' | 'destroyed'
earth.isDestroyed;
```

### 3.4 Earth 改为服务入口

2.0.0 的 Earth 统一提供：

```ts
earth.elements;
earth.layers;
earth.styles;
earth.animations;
earth.draw;
earth.transform;
earth.measure;
earth.events;
earth.contextMenu;
earth.overlays;
earth.view;
earth.controls;
```

常用迁移映射：

| 1.x | 2.0.0 |
| --- | --- |
| `earth.view`，原生 OL View | `earth.view.olView` |
| `earth.camera`、根导出的 `Camera` | `earth.view` |
| 根导出的 `Controls` | `earth.controls` |
| `earth.center` | 创建时传入 `view.center`；它只设置初始视图，不会改变 `flyHome()` 的内置目标 |
| `earth.containerId` | `earth.target`，返回值可能是字符串或 `HTMLElement` |
| `earth.flyHome()` | `earth.view.flyHome()` |
| `earth.animateFlyTo(center, zoom, duration)` | `earth.view.animateFlyTo(center, { zoom, duration })` |
| `earth.flyTo(center, zoom)` | `earth.view.flyTo(center, zoom)` |
| `earth.setMouseStyle(cursor)` | `earth.view.setCursor(cursor)` |
| `earth.setMouseStyleToDefault()` | `earth.view.useDefaultCursor()` |
| `earth.setMouseStyleToCrosshair()` | `earth.view.useCrosshairCursor()` |
| `earth.disabledMapDrag()` | `earth.view.setDragEnabled(false)` |
| `earth.enableMapDrag()` | `earth.view.setDragEnabled(true)` |
| `earth.graticule` | `earth.controls.graticule` |
| `earth.scaleLine` | `earth.controls.scaleLine` |
| `earth.enableGraticule()` | `earth.controls.enableGraticule()` |
| `earth.disableGraticule()` | `earth.controls.disableGraticule()` |
| `earth.enableScaleLine()` | `earth.controls.enableScaleLine()` |
| `earth.disableScaleLine()` | `earth.controls.disableScaleLine()` |
| `earth.getFeatureAtPixel(pixel)` | `earth.elements.atPixel(pixel)` |
| `earth.createOsmLayer()` + `earth.addLayer(layer)` | `earth.layers.add({ kind: 'tile', preset: 'osm' })` |
| `earth.createXyzLayer(urlOrFunction)` | `earth.layers.add()` 的 `xyz`、`compact-xyz` 或 `tileUrlFunction` 配置 |
| `earth.addLayer(olLayer)` | `earth.layers.add({ kind: 'native', layer: olLayer, ownership })` |
| `earth.removeLayer(layerOrId)` | 保存 V2 Layer 句柄或 ID，再调用 `layer.remove()` 或 `earth.layers.remove(id)` |
| `earth.registerLayer/getLayer/removeRegisteredLayer/listRegisteredLayers` | 直接使用 `earth.layers` 的 Layer ID、句柄和查询方法 |
| `earth.useDefaultLayer()`、`DefaultEntities.reset()` | 使用默认 vector 图层或自建图层，并通过 `layerId`、`module` Selector 管理元素 |

`earth.map` 仍然公开，类型仍是 OL `Map`。

`earth.center` 不再是可随时修改的 home 状态。V2 的 `view.center` 只设置初始视图；`earth.view.flyHome()` 当前返回引擎内置中心（默认投影下由经纬度 `[119, 39]` 转换）和 zoom 4，不读取创建时传入的 `view.center`。业务需要自定义 home 时，应自行保存目标坐标并调用 `earth.view.animateFlyTo()`。

V1 的 `addLayer()` 返回字符串句柄；V2 的 `earth.layers.add()` 返回公开 `Layer` 句柄。V1 的 `removeLayer()` 无参只删除带底图标记的图层，V2 没有这个快捷语义；不要用 `earth.layers.clear()` 直接替代，因为它会清除当前 Earth 管理的全部图层。

V1 `useDefaultLayer()` 中的 point、billboard、polyline、polygon 和 circle 统一迁到 `earth.elements`，overlay 迁到 `earth.overlays`，Wind 没有替代。V2 没有一个会同时清除这些能力的 `reset()`；应按业务保存 Layer/Element/Overlay 句柄，或使用明确的 `layerId`、`module` Selector 分别清理。

命中返回值也发生了变化：

```ts
// 1.x：始终返回对象，通过 isExists 判断
const legacyHit = earth.getFeatureAtPixel(pixel);
legacyHit.isExists;
legacyHit.id;
legacyHit.module;
legacyHit.feature;
legacyHit.layer;

// 2.0.0：未命中时返回 undefined
const hit = earth.elements.atPixel(pixel);
if (hit) {
  hit.element.state.id;
  hit.element.state.module;
  hit.element.olFeature;
  hit.layer;
}
```

`earth.elements.atPixel()` 只命中当前 Earth 管理的 `Element`。`kind: 'native'` 图层中的原生 Feature，以及业务直接通过 `earth.map` 加入的 Feature，不会出现在这个结果里；这类场景应继续使用 `earth.map.forEachFeatureAtPixel()` 等 OL 原生命中 API。

V1 的 `getLayerAtFeature(feature)` 没有按原生 Feature 反查公开 Layer 的直接替代。使用 V2 `Element` 时，可以通过 `element.state.layerId` 再调用 `earth.layers.get(layerId)`；使用原生图层时，应在业务侧保留 OL 图层引用。

## 4. 图层和 Element

### 4.1 删除按几何类型划分的图层类

以下 1.x 类已删除：

- `Base`
- `PointLayer`
- `BillboardLayer`
- `PolylineLayer`
- `PolygonLayer`
- `CircleLayer`
- `OverlayLayer`
- `WindLayer`

2.0.0 使用 `earth.layers` 管理渲染图层，使用 `earth.elements` 管理业务元素。一个 vector 图层可以同时承载点、图标点、线、面、圆和 Plot 图形。

普通 Polygon 有一个需要单独处理的不兼容点：1.x 的 `positions: Coordinate[][]` 可以直接表达外环、内环和洞；2.0.0 的内置 `polygon` 只接受一维 `controlPoints`，当前只生成单环。把外环和内环拆成多个单环元素会填满原来的洞，不能保持原语义；只有确认各环本来就是互不相干的独立面时才能拆分。需要保留洞时，应接入原生 OL 图层，或暂缓迁移这部分业务。

1.x：

```ts
const layer = new PointLayer(earth, { wrapX: false, register: false });
const feature = layer.add({
  id: 'vehicle-1',
  module: 'vehicles',
  center: [0, 0],
  size: 10,
  fill: { color: '#1677ff' }
});

layer.set({ id: 'vehicle-1', center: [100, 100] });
layer.hide('vehicle-1');
layer.remove('vehicle-1');
```

2.0.0：

```ts
const layer = earth.layers.add({
  kind: 'vector',
  id: 'business',
  wrapX: false,
  declutter: true,
  opacity: 1,
  zIndex: 10
});

const element = earth.elements.add({
  id: 'vehicle-1',
  module: 'vehicles',
  layerId: layer.id,
  geometry: { type: 'point', controlPoints: [[0, 0]] },
  style: {
    symbol: {
      type: 'circle',
      radius: 10,
      fill: { type: 'solid', color: '#1677ff' }
    }
  }
});

element.update({ geometry: { type: 'point', controlPoints: [[100, 100]] } });
earth.elements.hide({ id: element.id });
element.remove();
```

### 4.2 LayerService

创建矢量图层：

```ts
const vector = earth.layers.add({
  kind: 'vector',
  id: 'planning',
  visible: true,
  opacity: 0.9,
  zIndex: 20,
  wrapX: false,
  declutter: true
});
```

创建瓦片图层：

```ts
const osm = earth.layers.add({ kind: 'tile', preset: 'osm' });

const xyz = earth.layers.add({
  kind: 'tile',
  preset: 'xyz',
  url: 'https://example.com/{z}/{x}/{y}.png'
});

const callbackXyz = earth.layers.add({
  kind: 'tile',
  preset: 'xyz',
  tileUrlFunction: ([z, x, y]) => `/tiles/${z}/${x}/${y}.png`
});

const compact = earth.layers.add({
  kind: 'tile',
  preset: 'compact-xyz',
  baseUrl: '/compact-tiles'
});
```

`compact-xyz` 对应 1.x `createXyzLayer(baseUrl)` 的 `L{z}/R{y}/C{x}.jpg` 紧凑目录格式。

接入原生 OL 图层：

```ts
const native = earth.layers.add({
  kind: 'native',
  id: 'custom-native',
  layer: olLayer,
  ownership: 'external'
});
```

`ownership` 的含义：

- `'external'`：原生对象由业务管理，移除 Layer 时不销毁它。
- `'earth'`：原生对象交给当前 Earth 管理，移除或销毁时一并清理。

图层操作：

```ts
vector.update({ opacity: 0.6, zIndex: 30 });
vector.hide();
vector.show();

earth.layers.get(vector.id);
earth.layers.query('vector');
earth.layers.remove(vector.id);
earth.layers.clear();
```

常用旧图层封装方法迁移：

| 1.x | 2.0.0 |
| --- | --- |
| `base.setLayerOpacity(50)` | `layer.update({ opacity: 0.5 })` |
| `base.setLayerIndex(20)` | `layer.update({ zIndex: 20 })` |
| `base.getLayer()` | `layer.olLayer` |
| 几何图层的 `setPosition(id, coordinates)` | `element.update({ geometry: nextGeometry })` |
| `OverlayLayer.setPosition(id, position)` | `overlay.setPosition(position)` |
| `base.hide(id)` / `base.show(id)` | `earth.elements.hide({ id })` / `earth.elements.show({ id })` |
| `base.remove(id)` | `earth.elements.remove({ id })` 或 `element.remove()` |

注意透明度单位：V1 `setLayerOpacity()` 使用 `0` 到 `100` 的百分比；V2 `LayerPatch.opacity` 使用 `0` 到 `1`。直接把 `50` 传给 V2 不会得到半透明效果。

原生逃生口：

```ts
vector.olLayer;
```

### 4.3 ElementService 和 Selector

`Element` 是 2.0.0 的统一业务对象。`Element.state` 是业务状态真源：

```ts
const element = earth.elements.add({
  id: 'area-1',
  module: 'planning',
  layerId: 'planning-layer',
  geometry: {
    type: 'polygon',
    controlPoints: [
      [0, 0],
      [100, 0],
      [100, 100]
    ]
  },
  style,
  data: { name: '区域一' },
  visible: true
});
```

圆使用明确的圆心和半径，其中圆心仍是当前 View 的投影坐标，半径固定使用米：

```ts
const circle = earth.elements.add({
  geometry: {
    type: 'circle',
    center: earth.view.toProjectedCoordinates([120, 0]),
    radius: 1000
  }
});

const geometry = circle.state.geometry;
if (geometry.type !== 'circle') throw new Error('Expected circle geometry');
console.log(geometry.radius); // 1000，单位为米
```

`earth.view.toProjectedCoordinates()` 会把 EPSG:4326 经纬度转换到当前 Earth 的 View 投影。它接受单组或多组扁平坐标，也接受一层嵌套的二维或三维坐标，并保持输入结构。空数组、奇数长度的扁平数组、稀疏数组、混合结构和非有限数值会抛出 `InvalidArgumentError`。

`ShapeInput<'circle'>.radius`、`Element.state.geometry.radius`、元素快照、复制结果以及 Draw、Edit、Transform 的持久结果都使用米。`element.olFeature` 中原生 OL Circle 的半径仍是 View 投影单位，不要把它当成业务半径保存。`style.symbol.radius` 仍是 CSS 像素，与几何圆的半径无关。

元素写入接口同时接受扁平的二维控制点。扁平数组会按 `XY` 两两分组，`add()`、`update()`、`Element.update()`、`copy()` 和 Transform 复制使用相同规则：

```ts
const geographic = [120, 0, 110, 0];
const projected = earth.view.toProjectedCoordinates(geographic);

const line = earth.elements.add({
  geometry: {
    type: 'polyline',
    controlPoints: projected
  }
});

console.log(line.state.geometry);
// { type: 'polyline', controlPoints: [[13358338.89, 0], [12245143.99, 0]] }
```

扁平输入只简化数组结构，不会转换坐标投影。如果直接把 `[120, 0, 110, 0]` 传给 `elements.add()`，它仍会被当成当前 View 的投影坐标。外部业务保存经纬度时，写入元素前应显式调用 `earth.view.toProjectedCoordinates()`；从 `Element.state` 读取后应调用 `earth.view.toGeographicCoordinates()` 转回经纬度。三维坐标请使用 `[[x, y, z], ...]`，转换时 Z 值会原样保留。

保存成业务常用的扁平经纬度数组：

```ts
import { toFlatCoordinates } from '@vrsim/earth-engine-ol';

const element = earth.elements.get(line.id);
if (element?.state.geometry.type === 'polyline') {
  const geographicPoints = earth.view.toGeographicCoordinates(element.state.geometry.controlPoints);
  const saved = toFlatCoordinates(geographicPoints);
  console.log(saved); // 约为 [120, 0, 110, 0]
}
```

查询和批量操作统一使用 `ElementSelector`：

```ts
earth.elements.get('area-1');
earth.elements.query({ module: 'planning' });
earth.elements.query({ ids: ['a', 'b'], visible: true });
earth.elements.query({ layerId: 'planning-layer', type: 'polygon' });
earth.elements.query({ predicate: state => state.data !== undefined });

earth.elements.update({ module: 'planning' }, { visible: false });
earth.elements.show({ module: 'planning' });
earth.elements.remove({ module: 'planning' });
```

Selector 支持 `id`、`ids`、`module`、`layerId`、`type`、`visible` 和 `predicate`。

复制元素：

```ts
const copy = earth.elements.copy(element.id, {
  module: 'planning-copies',
  geometry: nextGeometry
});
```

1.x 的 `feature.get('param')`、`FEATURE_KEYS`、`LAYER_TYPE` 和从 OL Feature、Geometry、Style 反向恢复业务参数的行为已删除。持久修改必须更新 `Element.state`：

```ts
element.state;
element.update({ data: nextData });
```

`element.olFeature` 是高级 OL 逃生口。直接修改返回的 Feature 不会反向写回 `Element.state`，并可能在下一次状态投影时被覆盖。

### 4.4 Billboard 合并为图标点

1.x 的 `BillboardLayer` 已合并到 point Element：

```ts
const marker = earth.elements.add({
  geometry: { type: 'point', controlPoints: [[0, 0]] },
  style: {
    symbol: {
      type: 'icon',
      src: '/marker.png',
      size: [48, 48],
      anchor: [24, 48],
      anchorXUnits: 'pixels',

    }
  }

```

## 5. 样式

2.0.0 将样式从各类 Layer 参数中抽出，统一保存到 Element，并通过 `earth.styles` 更新。

### 5.1 参数映射

| 1.x | 2.0.0 |
| --- | --- |
| Point 的 `size`、`fill`、`stroke` | `symbol: { type: 'circle', radius, fill, stroke }` |
| Billboard 参数 | `symbol: { type: 'icon', ... }` |
| `label` | `text` |
| `stroke` + `backgroundStroke` | `strokes: [backgroundStroke, stroke]` |
| 纯色 `fill` | `{ type: 'solid', color }` |
| 纹理填充 | `{ type: 'pattern', pattern, ... }` |
| `isArrow` | `decorations: [{ type: 'arrow', placement: ... }]` |
| `isFlowingDash` | `earth.animations.play(..., { type: 'dash-flow' })` |

结构化样式示例：

```ts

```

图标锚点、偏移、缩放、旋转、透明度，文本字体、字号、颜色、背景、描边、padding、偏移和旋转，多层描边、纹理和箭头等能力均保留。旋转继续使用角度；文本 `offsetY` 正值继续表示向上偏移。

### 5.2 设置和局部更新

```ts
earth.styles.set({ id: element.id }, stylePresets['point-default']);

earth.styles.patch({ id: element.id }, {
  symbol: { radius: 12 },
  text: {
    fontSize: 18,
    backgroundFill: { type: 'solid', color: '#ffffff' }
  }
});
```

数组字段在 patch 中按整体替换处理；对象字段递归合并；显式 `undefined` 用于删除字段。

内置样式包括：

- `point-default`
- `icon-default`
- `line-default`
- `arrow-default`
- `polygon-default`
- `measure-default`
- `draw-preview`
- `transform-handle`

### 5.3 原生 OL 样式

需要直接使用 OL `StyleLike` 时：

```ts
earth.styles.set(
  { id: element.id },
  { nativeStyle: olStyle }
);
```

`nativeStyle` 支持 OL `Style`、`Style[]` 和 `StyleFunction`，但有明确限制：

- 不反向解析为 `StyleSpec`。
- 不能使用结构化 `StylePatch` 修改。
- 不支持任何 2.0 内置动画；动画管理器会在创建播放记录前拒绝 `NativeStyleRef`。
- 不承诺跨 OpenLayers 主版本兼容。

V1 根导出的 `createPatternFill()`、`drawPatternFill()`、`isPatternFill()` 和 `normalizePatternFill()` 已删除。业务侧不再直接创建 CanvasPattern；改为把 `PatternFillSpec` 交给 `StyleSpec.fill`、`symbol.fill` 或文字填充，由样式编译器统一处理。

## 6. Draw、Edit 和 Plot

### 6.1 统一为 DrawService

1.x：

```ts
const draw = earth.useDrawTool();

draw.drawPoint({ callback });
draw.drawAttackArrow({ callback });
draw.modifyPolygon({ feature, callback });
```

2.0.0：

```ts
const session = earth.draw.start({
  type: 'attack-arrow',
  layerId: 'planning-layer',
  module: 'planning',
  style,
  limit: 1,
  keepGraphics: true,
  policy: 'replace'
});

session.on('complete', ({ element }) => {
  console.log(element.state);
});
```

不再为每一种图形提供独立方法。`type` 统一支持 20 种图形：

```text
point
polyline
polygon
circle
ellipse
attack-arrow
tailed-attack-arrow
fine-arrow
tailed-squad-combat-arrow
assault-direction-arrow
double-arrow
rectangle
triangle
equilateral-triangle
assemble-polygon
closed-curve-polygon
sector
lune-polygon
lune-polyline
curve-polyline
```

### 6.2 Plot 不再是独立入口

1.x 根入口实际只暴露过 `PlotDrawEventName`、`PlotDrawListener` 等类型；`PlotDraw`、`PlotEdit` 和 `TransformInteraction` 默认类通常来自 `dist/*` 或历史深导入，并不是稳定的根运行时导出。2.0.0 删除了这些低层入口。Plot 图形已经是普通 `ShapeType`，与点、线、面使用同一套绘制、编辑、Transform、样式和 Element 生命周期。

当前版本不开放自定义 `ShapeDefinition` 注册接口。

### 6.3 绘制会话

```ts
session.status;
session.results;
await session.finished;

session.undo();
session.redo();
session.finish();
session.cancel();
session.destroy();
```

可订阅 `start`、`change`、`click`、`complete` 和 `cancel`。旧事件名不能直接沿用：

| 1.x | 2.0.0 |
| --- | --- |
| `drawstart` | `start` |
| `drawing` | `change` |
| `drawingClick` | `click` |
| `drawend` | `complete` |
| `drawexit` | `cancel` |
| `modifying` | 编辑会话的 `modifying` |
| `modifyexit` | 根据结果改用编辑会话的 `complete` 或 `cancel` |

V1 回调主要返回 OL Feature 和坐标字段；V2 的 `change` 返回 `ShapeState`，`complete` 返回公开 `Element`，编辑事件也通过 `Element` 和工作几何传递结果。右键会退出当前绘制草图。

V1 Draw 的坐标单位并不统一：`drawstart`、`drawend` 以及点、线、面结果中的 `featurePosition` 通常是 View 投影坐标；`drawing`、`drawingClick`、退出事件和 Circle 的 `featurePosition` 则经过经纬度转换。V2 的事件坐标和 `ShapeState` 坐标全部使用当前 View 投影，不再自动混用经纬度；圆的 `ShapeState.radius` 则固定使用米。迁移旧回调时，应逐个确认原字段语义；需要经纬度时显式调用 `earth.view.toGeographicCoordinates()`。

`keepGraphics: false` 时，完成元素只在同步 `complete` 回调期间有效，监听器执行完后会被移除，也不会进入 `results`。

`earth.draw.query()` 和 `earth.draw.clear()` 只处理 DrawService 自己创建且仍存在的元素，不会误删其他来源的元素。

### 6.4 编辑会话

```ts
const edit = earth.draw.edit(element, {
  underlay: true,
  policy: 'replace'
});

edit.on('modifying', event => {
  console.log(event.geometry, event.operation);
});

edit.undo();
edit.redo();
edit.finish();
```

编辑会话提供 `modifying`、`complete` 和 `cancel` 事件。右键提交当前编辑。目标在编辑期间被外部更新或删除时，会话会取消，不会覆盖外部结果。

方法映射：

| 1.x | 2.0.0 |

| `earth.useMeasure().lineSegmentation()` | `earth.measure.start({ type: 'distance-segments' })` |
| `lineFirst()` | `earth.measure.start({ type: 'distance-total' })` |
| `lineCenter()` | `earth.measure.start({ type: 'distance-radial' })` |
| `polygonMeasure()` | `earth.measure.start({ type: 'area' })` |
| 配置中的 `callback` | `session.on('change' | 'complete', callback)` |

示例：

```ts
const session = earth.measure.start({
```

## 8. Transform

### 8.1 创建方式

1.x：

```ts
const transform = new Transform({
  earth,
  translateType: ETranslateType.Feature,
  scale: true,
  stretch: true,
  rotate: true,
  transformLayers: [layer.getLayer()],
  beforeTransform: feature => true,
  historyLimit: 10
});

transform.on(ETransform.TranslateEnd, callback);
```

2.0.0 可以先创建等待选择的会话：

```ts
const session = earth.transform.start({
  selector: { module: 'planning' },
  layerIds: ['planning-layer'],
  translate: 'feature',
  scale: true,
  stretch: true,
  rotate: true,
  historyLimit: 10,
  toolbar: true,
  policy: 'replace'
});

session.select(element);
```

也可以直接选择元素：

```ts
const session = earth.transform.select(element, options);
```

过滤方式改为：

- `selector`：按 id、module、layerId、type 等业务状态筛选。
- `predicate`：接收公开 `Element`，不再接收 OL Feature。
- `layerIds`：按 V2 Layer ID 限制图层。

`translateType` 枚举改为 `translate: 'none' | 'center' | 'feature'`。

### 8.2 会话操作和事件

```ts
session.selected;
session.status;
session.mode;
session.toolbar;

session.setMode('transform');
session.setMode('edit');
session.undo();
session.redo();
session.copy();
session.replaceSelected(nextElement, { retainHistory: true });
session.remove();
session.finish();
session.cancel();
```

1.x 的 `replaceEditingFeature()` 改为 `replaceSelected()`。旧 `destroy()` 改为按业务意图调用 `finish()` 或 `cancel()`；Earth 销毁时会统一清理仍活动的 Transform 会话。

事件枚举改为字符串事件名：

```ts
session.on('select', event => console.log(event.element));
session.on('translateEnd', event => console.log(event.element));
session.on('rotateEnd', event => console.log(event.element));
session.on('scaleEnd', event => console.log(event.element));
session.on('remove', event => console.log(event.element));
```

完整事件名迁移：

| 1.x | 2.0.0 |
| --- | --- |
| `select` | `select` |
| `selectend` | `selectEnd` |
| `enterHandle` / `leaveHandle` | `enterHandle` / `leaveHandle` |
| `translatestart` / `translating` / `translateend` | `translateStart` / `translating` / `translateEnd` |
| `rotatestart` / `rotating` / `rotateend` | `rotateStart` / `rotating` / `rotateEnd` |
| `scalestart` / `scaling` / `scaleend` | `scaleStart` / `scaling` / `scaleEnd` |
| `modifystart` | 无一一对应事件；由业务在 `setMode('edit')` 时记录进入编辑 |
| `modifying` | 顶点变化时触发 `edit` |
| `modifyend` | 无一一对应事件；根据会话的 `finish()` 或 `cancel()` 处理 |
| `copy` | `copyPreviewConfirm`；直接复制和预览确认成功都会触发，`copyPreviewCancel` 只用于新增的预览取消场景 |
| `remove` | `remove` |
| `undo` / `redo` 事件 | 无对应事件；根据 `undo()` / `redo()` 返回值处理 |

V1 `on()` 可以一次传入事件数组，并通过 `off()` 移除；V2 每次订阅一个事件，必须保存 `session.on()` 返回的幂等注销函数：

```ts
const off = session.on('translateEnd', callback);
off();
```

V2 Transform 事件载荷以公开 `Element` 为主，不再把 OL Feature 作为主要业务对象，并新增 `error` 事件用于接收会话捕获的失败。

V2 将普通变换和顶点编辑明确分成两种模式。只有进入 `edit` 模式后才显示编辑控制点。

### 8.3 行为变化

- 删除公开的低层 `TransformInteraction` 类。
- Transform 只操作当前 Earth 的元素，多实例之间互不影响。
- 右键优先由活动 Transform 会话处理，不会误打开业务菜单。
- 变换预览会与统一动画管理器同步。
- 缩放几何时，文本、描边、图标参数、纹理和装饰等屏幕样式不会被错误放大。
- 平移或旋转几何圆不改变米制 `geometry.radius`；缩放圆时按缩放比例修改该米制半径。
- 点元素的 circle 半径和 icon scale 可按目标能力变换。
- NativeStyle 点元素的视觉缩放或旋转无法安全同步时会拒绝操作。
- Transform 会把点图标旋转手势的投影坐标弧度增量换算为 `StyleSpec` 使用的顺时针角度，预览、撤销、重做和最终提交保持一致；普通几何旋转继续写回 geometry state。

## 9. 事件和右键菜单

### 9.1 EventService

1.x：

```ts
const events = earth.useGlobalEvent();
const dispose = events.addMouseClickEventByGlobal(callback);
events.addMouseClickEventByModule('vehicle', callback);
events.disableGlobalMouseClickEvent();
```

2.0.0：

```ts
const dispose = earth.events.on('click', event => {
  console.log(event.coordinate, event.pixel, event.element);
});

const disposeVehicle = earth.events.on(
  'click',
  event => console.log(event.element),
  { module: 'vehicle' }
);

const cancelOnce = earth.events.once('rightclick', callback);
```

不再需要手动 enable/disable。`on()` 和 `once()` 注册时自动启用，并返回幂等注销函数。也可以通过 `AbortSignal` 清理：

```ts
const controller = new AbortController();
earth.events.on('pointermove', callback, { signal: controller.signal });
controller.abort();
```

事件坐标的语义发生了变化。V1 `GlobalEvent` 回调中的 `position` 已经经过 `toLonLat()`，通常是经纬度；V2 的 `coordinate` 是当前 View 投影下的地图坐标。若业务仍需要经纬度，使用当前 Earth 实例显式转换：

```ts
earth.events.on('click', event => {
  const lonLat = earth.view.toGeographicCoordinates(event.coordinate);
  console.log(lonLat);
});
```

V1 的 keydown 回调直接收到 `KeyboardEvent`；V2 收到 `EarthKeyboardEvent` 包装对象，常用字段已提升为 `key`、`code` 和修饰键状态，原生事件位于 `originalEvent`，且只保证在同步回调期间有效。

指针事件支持 `selector` 或 `module`，两者不能同时传。`keydown` 只能注册为 Earth 全局事件。

其他管理方法：

```ts
earth.events.has('click');
earth.events.has('click', 'vehicle');
earth.events.clearModule('vehicle');
earth.events.clearModule('vehicle', 'click');
```

### 9.2 ContextMenuService

1.x：

```ts
earth.useContextMenu().addDefaultMenu(items, callback);
earth.useContextMenu().addModuleMenu('vehicle', items, callback);
```

2.0.0：

```ts
const mapMenu = earth.contextMenu.register('map', {
  items: [{ key: 'inspect', label: '查看位置' }],
  before: context => true,
  onSelect: context => console.log(context.coordinate)
});

const moduleMenu = earth.contextMenu.register(
  { module: 'vehicle' },
  {
    items: [{ key: 'locate', label: '定位车辆' }],
    onSelect: context => console.log(context.element)
  }
);

const elementMenu = earth.contextMenu.register(element, spec);
```

右键菜单回调的坐标与事件服务相同：V1 `position` 是经纬度，V2 `coordinate` 是当前 View 投影坐标，需要经纬度时请使用 `earth.view.toGeographicCoordinates()` 显式转换。

常用字段迁移：

| 1.x | 2.0.0 |
| --- | --- |
| 菜单项 `child` | `children` |
| 回调 `scope: 'default'` | `scope: 'map'` |
| 回调 `menu` | `item` |
| 回调 `featureId` | `element?.state.id` |
| 回调 `feature` | `element?.olFeature` |
| 回调 `param` | 按新结构从 `element?.state` 的 id、module、geometry、style、data 等字段读取 |
| 回调 `param.data` | `element?.state.data` |
| `earth.clearContextMenuState(module, featureId)` | `earth.contextMenu.clearElementState(featureId)` |

每次注册返回独立句柄：

```ts
mapMenu.destroy();
moduleMenu.destroy();
elementMenu.destroy();
```

菜单项支持嵌套、显隐、禁用和互斥状态：

```ts
earth.contextMenu.getItemState(element, 'locate');
earth.contextMenu.setItemState(element, 'locate', { disabled: true });
earth.contextMenu.toggleItem(element, 'locate');
earth.contextMenu.setTheme('dark');
earth.contextMenu.close();
```

浏览器原生右键菜单仍会被屏蔽。屏蔽与右键交互仲裁归当前 Earth 的输入路由管理，默认实例和多个命名实例之间不会互相释放或覆盖监听。

## 11. 动画

1.x 的点闪烁、流水线和飞行线分别由不同 Layer 管理：

```ts
pointLayer.set({ id, isFlash: true, duration: 900 });
pointLayer.stopFlash(id);

polylineLayer.add({ id, isFlowingDash: true });
polylineLayer.addFlightLine(options);
```

2.0.0 统一使用 `earth.animations.play(selector, spec)`；不再按点、线、区域或雷达拆分专用动画 Manager。

点脉冲：

```ts
const pulse = earth.animations.play(
  { id: pointId },
  {
    type: 'pulse',
    periodMs: 900,
    color: '#ff6347',
    radius: 8,
    repeat: true
  }
);
```

虚线流动：

```ts
const flow = earth.animations.play(
  { id: lineId },
  {
    type: 'dash-flow',
    speed: 24,
    lineDash: [10, 18],
    color: '#ffffff'
  }
);
```

V1 流水线的 `fullLineColor` 是静态底线颜色，`dottedLineColor` 是流动虚线颜色。V2 应按背景在前、流动前景在后的顺序写入两层 `strokes`：第一层使用 `fullLineColor`，最后一层使用 `dottedLineColor`。`dash-flow` 只给最后一层描边设置流动偏移，前面的描边继续作为静态轮廓。

路径飞行：

```ts
const travel = earth.animations.play(
  { id: lineId },
  {
    type: 'path-travel',
    durationMs: 2500,
    repeat: false,
    trailLength: 0.35,
    gradient: [
      [0, '#38bdf8'],
      [1, '#f97316']
    ],
    width: 5,
    curvature: 0.45,
    showStart: true,
    showEnd: true,
    finishBehavior: 'retain'
  }
);
```

2.0.0 还在同一 `play()` 入口下新增七种可组合效果：

```ts
earth.animations.play({ id: areaId }, { type: 'blink' });
earth.animations.play({ id: areaId }, { type: 'highlight', mode: 'breathe' });
earth.animations.play({ id: areaId }, { type: 'alert' });
earth.animations.play({ id: routeId }, { type: 'grow', durationMs: 1200 });
earth.animations.play({ id: sectorId }, { type: 'radar-scan' });
earth.animations.play({ id: sectorId }, { type: 'center-spread' });
earth.animations.play({ id: elementId }, { type: 'fade', direction: 'out' });
```

- `blink` 和 `fade` 支持所有使用 `StyleSpec` 的结构化 Shape。
- `highlight` 和 `alert` 支持非退化闭合面。
- `grow` 支持 Polyline、LunePolyline、CurvePolyline 和内置面箭头 provider。
- `radar-scan` 和 `center-spread` 支持 Circle 与 Sector。两者都默认使用 `#00e676` 绿色效果，并且纯色 `color` 与多段 `gradient` 互斥。
- `radar-scan` 的 gradient offset `0` 表示尾迹最旧端、`1` 表示扫描前沿；`center-spread` 的 offset `0` 表示内侧最旧尾迹、`1` 表示外侧波纹前沿。
- `center-spread` 默认 `opacity` 为 `0.7`、`trailLength` 为 `0.18`；将 `trailLength` 设为 `0` 可保留旧版纯线环视觉。
- 上述七种效果以及原有 `pulse`、`dash-flow`、`path-travel` 都不支持 `NativeStyleRef`；批量播放中任一目标使用 native style 时，整次 `play()` 会在建立记录前抛出 `UnsupportedOperationError`。

完整兼容矩阵、Spec 字段和 retain 语义见 Animation 动画效果页。

常用旧参数迁移：

- Point 的 `duration` 改为 `pulse.periodMs`。
- Point 的 `flashColor: { R, G, B }` 改为 CSS 颜色字符串、RGB 数组或 RGBA 数组，例如 `color: [255, 99, 71]`。
- Polyline 的 `arrowIsRepeat: true` 对应 `decorations: [{ type: 'arrow', placement: 'each-segment' }]`；V2 的 `placement: 'repeat'` 是按像素间距重复，不是旧语义。
- FlightLine 的 `splitLength`、`oneFrameLimitTime`、`controlRatio` 和径向颜色配置，需要重新映射到 `smoothness`、`durationMs` 或 `speed`、`curvature`、`gradient`。这些字段的单位和计算方式不同，不能直接复制数值。
- `addFlightLine()` 改为创建线 Element 后调用 `earth.animations.play(..., { type: 'path-travel' })`；`setFlightPosition()` 改为更新 Element geometry。V1 `removeFlightLine()` 会移除飞行线及其辅助内容；V2 停止 `AnimationHandle` 只清理动画渲染，不会删除作为目标的线 Element。需要同样的删除结果时，应先 `handle.stop()`，再 `element.remove()`。
- `PathTravelAnimationSpec` 不再接受 `arrow` 和 `arrowColor`，路径运动只绘制尾迹与可选起终点。静态方向箭头使用 Element 的结构化 Decoration，箭头生长使用 `grow`；严格校验会把旧字段作为未知成员拒绝。

单动画控制：

```ts
pulse.pause();
pulse.resume();
pulse.stop();
await pulse.finished;
```

按 Selector 和 channel 批量控制：

```ts
earth.animations.pause({ module: 'vehicles' });
earth.animations.resume({ module: 'vehicles' }, ['status']);
earth.animations.stop({ layerId: 'routes' });
earth.animations.stopAll();
```

同一元素、同一 channel 的新动画会在完整校验成功后原子替换旧动画。不同 channel 只在写入域可组合时组合：`blink` 与 `fade` 的 target-opacity 按乘法合成，overlay 效果稳定追加，`grow` 独占 target-geometry。两个不同 channel 的 `grow` 会原子抛出 `CapabilityError`，不会静默覆盖或部分播放。动画运行状态不会写入 Element 快照，也不会随元素复制。

`AnimationType`、`AnimationSpec` 判别联合和公开 `animationTypes` 可能在后续功能版本的末尾追加内置成员。除上述 `path-travel` 箭头字段移除外，旧成员的字段、默认值、顺序和行为保持稳定；消费方的 exhaustive switch、`assertNever` 或 `Record<AnimationType, ...>` 必须保留未知成员兜底，不应假定联合永久只有当前十项。

V2 已修复 1.x 单条 dash-flow 移除和 FlightLine postrender 监听无法完整清理的问题。

## 12. Utils 和旧枚举

### 12.1 Utils 拆分

| 1.x | 2.0.0 |
| --- | --- |
| `Utils.GetGUID()` | `createId()` |
| `Utils.linearInterpolation(start, end, t)` | `lerp2(start, end, t)` |
| `Utils.constantMultiVector2(factor, vector)` | `scale2(vector, factor)`，参数顺序反转 |
| `Utils.vector2Add(a, b)` | `add2(a, b)` |
| `Utils.bezierSquareCalc(start, control, end, t)` | `quadraticBezier2(start, control, end, t)` |
| `Utils.ensureClosedRing(ring)` | `closeRing(ring)` |
| `Utils.trimClosedRing(ring)` | `trimClosingCoordinate(ring)` |
| `Utils.deg2rad(value)` | `degToRad(value)` |
| `Utils.rad2deg(value)` | `radToDeg(value)` |
| `Utils.getWorldWidth(map)` | `earth.view.worldWidth()` |
| `Utils.getWorldIndex(map, x)` | `earth.view.worldIndex(x)` |
| `Utils.normalizeToViewWorld(map, coordinates)` | `earth.view.normalizeToViewWorld(coordinates)` |
| `Utils.restoreToWorldIndex(map, coordinates, index)` | `earth.view.restoreToWorld(coordinates, index)` |
| `Utils.getFeatureToPixel(map, pixel, coordinates)` | `earth.view.translateCoordinatesToPixel(pixel, coordinates)` |
| 逐点 `fromLonLat()` | `earth.view.toProjectedCoordinates(coordinates)` |
| 逐点 `toLonLat()` | `earth.view.toGeographicCoordinates(coordinates)` |
| `Utils.flash(...)` | `earth.animations.play(selector, { type: 'pulse' })` |
| `Utils.createStyle(...)` | `StyleSpec` 或 `stylePresets['arrow-default']` |
| `Utils.throttle(...)` | 根导出的 `throttle(...)` |

V2 新增根导出的 `toFlatCoordinates()`，用于把读取到的嵌套坐标转换成适合业务保存的一维数组。它只展开数组，不修改输入，也不转换投影：

```ts
import { toFlatCoordinates } from '@vrsim/earth-engine-ol';

const saved = toFlatCoordinates([
  [120, 0],
  [110, 0]
]);
// [120, 0, 110, 0]
```

外部系统以经纬度保存坐标时，应先使用 `earth.view.toGeographicCoordinates()` 把 `Element.state` 中的 View 投影坐标转回 EPSG:4326，再按需要使用 `toFlatCoordinates()` 展平。反向写入时，使用 `earth.view.toProjectedCoordinates()` 把经纬度转成所属 Earth 的 View 投影坐标。两个方法都返回新数组，支持扁平二维数组和一层嵌套的二维或三维坐标；不要传入空数组。

V1 `Utils.getFeatureToPixel()` 对空值或无效输入会宽松地返回 `null`。V2 `earth.view.translateCoordinatesToPixel()` 在像素位置没有对应坐标或坐标数组为空时返回 `undefined`；像素或坐标结构不合法时会抛出 `InvalidArgumentError`，迁移时需要先校验输入，或在调用处捕获错误。

V1 `Utils.throttle()` 的默认等待时间是 `100ms`，V2 根导出的 `throttle()` 默认是 `0ms`。要保持旧节流频率，必须显式传入 `100`：

```ts
const onMove = throttle(callback, 100);
```

`createId()` 固定返回带连字符的 UUID。1.x 如果使用 `Utils.GetGUID('N')` 获取无连字符结果，需要在业务层明确转换：

```ts
const compactId = createId().replaceAll('-', '');
```

数学工具改为根导出的纯函数：

```ts
import {
  add2,
  closeRing,
  degToRad,
  lerp2,
  quadraticBezier2,
  radToDeg,
  scale2,
  trimClosingCoordinate
} from '@vrsim/earth-engine-ol';
```

旧 `DrawType`、`ModifyType`、`ETransform`、`ETranslateType`、`EPlotType`、`ECursor` 和 `EntityType` 等枚举不再公开。

`ECursor` 的值本来就是 CSS cursor 字符串，改为直接传给 `earth.view.setCursor('pointer')`。`EntityType` 中的 Point、Polyline、Polygon、Circle 分别改为小写 `ShapeType`，Rect 可在确认原有语义后改为 `rectangle`，Billboard 改为 point + icon 样式；Label、Heatmap、Model、MilStd 和 Custom 没有可以直接照搬的独立 `ShapeType`。

| `circle` | `circle` |
