# @vrsim/earth-engine-ol 1.x 升级到 2.0.0

2.0.0 调整了包导出、样式入口和 Earth 实例管理。升级现有外部项目时，请完成以下迁移；后文保留 0.x 升级到 1.0.0 的历史指南。

## 1. JavaScript 入口改为仅 ESM

2.0.0 不再提供 CommonJS/CJS 或 `require` 入口。请使用支持 ESM 的运行时或构建工具，并将调用改为标准 ESM import：

```ts
import { PointLayer, useEarth } from '@vrsim/earth-engine-ol';
```

不要再通过 `require('@vrsim/earth-engine-ol')` 加载本包。公开 JavaScript 入口均映射到 `.mjs` 文件。

## 2. 使用公开样式入口

1.x 的 `dist/index*.css` 深路径不再属于公共接口。统一改为：

```ts
import '@vrsim/earth-engine-ol/style.css';
```

包不再开放 `./dist/*` 深路径。除包根外，可按需从 `/core`、`/layers`、`/draw`、`/measure`、`/transform` 和 `/plot` 导入。

## 3. useEarth 获取或创建实例

1.x 的入口签名是 `useEarth(viewOptions?, options?)`；2.0 必须改为单个 UseEarthOptions 对象：`useEarth({ view, target, controls })`。2.0 运行时只读取第一个参数，因此旧两参调用的第二个参数会被忽略，原来直接放在第一参的 center、zoom 等视图字段也不会生效。

```ts
// 1.x
useEarth({ center, zoom }, { target: 'map', zoom: true });

// 2.0
useEarth({
  view: { center, zoom },
  target: 'map',
  controls: { zoom: true }
});
```

- `useEarth()` 获取或创建默认实例，默认挂载目标为 `olContainer`。
- `useEarth(id)` 获取或创建对应命名实例；首次创建时，非空 id 同时作为默认挂载目标。
- `useEarth(options)` 通过可选 id 选择默认或命名注册键，并配置挂载目标、视图与控件。

同一注册键已有活动实例时，上述调用都会返回原对象；target、view 和 controls 仅在首次创建时生效。多地图场景下，图层和工具内部会显式使用其所属 Earth，外部仍可通过 id 便捷取回实例。

```ts
const defaultEarth = useEarth({ target: 'map' });
console.assert(useEarth() === defaultEarth);

const overview = useEarth({ id: 'overview', target: 'overview' });
console.assert(useEarth('overview') === overview);
```

## 4. 销毁、注销与重建

可以直接调用 `earth.destroy()`，也可以使用公开辅助函数：

```ts
import { destroyEarth } from '@vrsim/earth-engine-ol';

destroyEarth(); // 销毁默认实例
destroyEarth('overview'); // 销毁命名实例
```

`destroyEarth()` 和 `destroyEarth(id)` 在不存在对应实例时不会抛错。销毁会同时注销注册键；之后以相同 key 调用 `useEarth()`、`useEarth(id)` 或 `useEarth(options)` 会创建新实例。

## 5. 依赖清理

2.0 移除了本库未使用的直接依赖 `heatmap.js`、`mitt` 和 `@types/heatmap.js`。业务直接使用这些包时需自行显式安装，不要依赖传递安装；其中 `@types/heatmap.js` 应按业务 TypeScript 配置作为开发依赖安装。

## 6. 2.0 升级 Checklist

- [ ] 将 CommonJS `require` 改为 ESM import，并确认运行时或构建工具支持 ESM。
- [ ] 将 `dist/index*.css` 替换为 `@vrsim/earth-engine-ol/style.css`。
- [ ] 移除所有 `./dist/*` 深路径导入，改用包根或公开功能子路径。
- [ ] 确认同一 id 的配置仅用于首次创建；需要应用新配置时先销毁原实例。
- [ ] 在组件卸载时调用 `earth.destroy()`、`destroyEarth()` 或 `destroyEarth(id)`。
- [ ] 如果业务直接导入 `heatmap.js`、`mitt` 或 `@types/heatmap.js`，在业务项目中显式声明对应依赖。
- [ ] 执行 `npm run verify` 与业务项目构建。

# @vrsim/earth-engine-ol v1.0.0 迁移指南

本库由 0.x 升级到 1.0.0，包含破坏性变更。外部消费端升级时请逐条核对。

## 1. 导入路径与符号重命名

旧名称已移除，不保留 deprecated 别名。

| 旧写法             | 新写法             |
| ------------------ | ------------------ |
| `commponents`      | `components`       |
| `Transfrom`        | `Transform`        |
| `ETransfrom`       | `ETransform`       |
| `ITransfromParams` | `ITransformParams` |

示例：

```ts
import { Transform, ETransform, ITransformParams } from '@vrsim/earth-engine-ol';
```

## 2. Utils 方法显式接收 map

以下方法不再依赖全局 `useEarth()` 单例：

| 旧方法                                     | 新方法                                          |
| ------------------------------------------ | ----------------------------------------------- |
| `Utils.getWorldWidth()`                    | `Utils.getWorldWidth(map)`                      |
| `Utils.getWorldIndex(x)`                   | `Utils.getWorldIndex(map, x)`                   |
| `Utils.getFeatureToPixel(pixel, position)` | `Utils.getFeatureToPixel(map, pixel, position)` |
| `Utils.normalizeToViewWorld(coords)`       | `Utils.normalizeToViewWorld(map, coords)`       |
| `Utils.restoreToWorldIndex(coords, idx)`   | `Utils.restoreToWorldIndex(map, coords, idx)`   |

```ts
const map = earth.map;
const next = Utils.getFeatureToPixel(map, pixel, baseCoords);
```

## 3. Utils.flash 静态化

```ts
// 旧
new Utils().flash(feature, param, layer);

// 新
Utils.flash(map, feature, param, layer);
```

## 4. Utils.GetGUID format 收窄

`Utils.GetGUID()` 现在只支持：

- `D`：带连字符，默认值。
- `N`：不带连字符。

`B`、`P`、`X` 原本未完整实现，已移除。

## 5. ol 改为 peerDependency

业务项目必须自行安装 OpenLayers：

```bash
npm install ol@^7
```

## 6. Earth 控件模块化

`earth.graticule` 和 `earth.scaleLine` 现在是只读 getter，透传自：

```ts
earth.controls.graticule;
earth.controls.scaleLine;
```

新增公共模块：

```ts
earth.camera;
earth.controls;
```

## 7. getUpdatedParam 返回类型收窄

`Base.getUpdatedParam(feature)` 从 `any` 收窄为：

```ts
AnyParam | undefined;
```

使用时建议按字段做类型收窄：

```ts
const param = layer.getUpdatedParam(feature);
if (param && 'positions' in param) {
  // polyline 或 polygon
}
```

## 8. 默认 Earth 实例访问调整

基础图层和标绘模块不再直接反向依赖 `useEarth.ts`，而是通过内部默认实例 provider 获取单例，以减少构建循环依赖。

多地图场景仍建议显式传入实例：

```ts
const layer = new PointLayer(earth);
```

## 9. 行为修复与优化

- `Earth.removeLayer()` 无参分支改为反向遍历，避免删除相邻底图时漏删。
- `Base.hide(id)` / `show(id)` 保留 change 监听，避免重复绑定。
- `Base.destroy()` 清理 change 监听、闪烁监听和隐藏缓存。
- feature change 同步默认使用约 33ms 节流，降低拖拽过程中的同步开销。
- 飞线不再反向创建 `PolylineLayer`，由 `PolylineLayer` 注入锚点线管理能力，避免循环依赖。

## 11. 生命周期与临时图层调整

本次调整以多 Earth 实例并存时的隔离性为目标，现有业务调用无需修改；如果项目自行管理全局事件或创建临时图层，请关注以下行为。

### 11.1 全局事件按实例释放

`DynamicDraw`、`Transform` 与 `Measure` 不再调用 `disableGlobal*` 关闭整类全局事件，而是仅释放自身注册的回调。

- 一个工具退出、销毁或重新开始时，不会影响同一 Earth 上其他工具的鼠标和键盘监听。
- `DynamicDraw` 连续启动绘制或编辑时，会主动关闭上一绘制/编辑会话的临时监听与资源。
- `Measure.lineCenter()` 的延迟左键抬起监听会在测量退出时取消，不会在退出后重新注册。

若业务代码仍直接调用 `enableGlobal*` / `disableGlobal*`，这些 API 仍保持兼容；多个功能并存时，推荐保留 `addMouse*EventByGlobal`、`addKeyDownEventByGlobal` 返回的释放函数，并在功能结束时调用。

### 11.2 可取消的一次性全局事件

新增以下方法，用于在一次性事件触发前主动取消监听：

```ts
const cancelClick = earth.useGlobalEvent().addCancelableMouseOnceClickEventByGlobal((event) => {
  // 左键确认
});

const cancelRightClick = earth.useGlobalEvent().addCancelableMouseOnceRightClickEventByGlobal((event) => {
  // 右键取消
});

// 会话提前结束时调用
cancelClick();
cancelRightClick();
```

原有的 `addMouseOnceClickEventByGlobal` 与 `addMouseOnceRightClickEventByGlobal` 仍可继续使用；它们保持原来的 `void` 返回类型。

### 11.3 临时图层不登记到 Earth

`PointLayer`、`PolylineLayer`、`PolygonLayer` 的构造参数新增可选 `register`：

```ts
const temporaryLayer = new PointLayer(earth, {
  wrapX: false,
  register: false
});
```

`register: false` 会将图层添加到 OpenLayers 地图，但不会登记到 `earth.getLayer()` 使用的图层注册表。该选项适用于编辑控制点、预览和其他生命周期短的内部图层；默认值仍为 `true`，普通业务图层不受影响。

### 11.4 Transform 多 Earth 与稳定控制框 API

Transform 交互现在使用创建它的 Earth 实例检查动态绘制和网格图层，不再依赖默认 Earth。多地图场景请继续显式传入 Earth：

```ts
const transform = new Transform({ earth });
```

对于需要访问自定义 TransformInteraction 控制框的扩展代码，请使用以下稳定方法，不再读取 `bbox_` 或调用 `drawSketch_`：

```ts
const bbox = interaction.getBoundingBoxFeature();
interaction.refreshSketch();
```

### 11.5 依赖清理

项目移除了未使用的直接依赖：`lodash.clonedeep`、`vue-tsc` 与 `@types/minimatch`。业务项目不需要额外操作；锁文件中如果仍存在 `@types/minimatch`，它可能来自其他包的传递依赖。
