# ContextMenu 地图联动示例设计

## 目标

保留 ContextMenu 文档的 API 主题和现有 13 个示例结构，把菜单回调从文字反馈升级为真实地图交互。页面说明只描述“在哪里右键、选择什么、地图发生什么”，不新增 Base、PointLayer、Earth 等辅助 API 的调用说明。

## 设计原则

- 每个示例继续使用独立、完整的 Vue 组件，预览与展示源码引用同一文件。
- 全局菜单只操作地图位置、临时标记或全局辅助图层；模块菜单只操作命中的车辆及其轨迹、告警等模块资源。
- 每组菜单至少包含三个业务相关项；菜单点击后必须出现可观察的地图变化，文字反馈仅作为辅助。
- 复用现有公开 API，不修改 ContextMenu、Base 或 Earth 的公开行为。

## 十三个示例

1. 最小生命周期：添加临时标记、清除标记、恢复初始视图。
2. 添加全局菜单：添加任务点、添加警戒点、清空全部标记。
3. 全局菜单点击回调：保存右键位置、移动观察中心、生成坐标标牌。
4. 添加模块菜单：定位车辆、切换车辆可见性、移除车辆。
5. 模块菜单守卫：有权限时移动或删除车辆，无权限时相关项禁用。
6. 模块菜单点击回调：针对命中车辆生成任务点、状态标记或调度结果。
7. 多级子菜单：车辆、轨迹、告警三个操作族分别联动对应图层。
8. 互斥菜单项：显示/隐藏车辆标签，开启/关闭轨迹层。
9. 菜单项显示隐藏：菜单项状态与对应地图功能可用性同步展示。
10. 菜单项状态切换：两辆车分别控制轨迹或告警状态，按 featureId 隔离。
11. 主题切换：在多对象地图场景中观察深浅菜单主题，不扩展主题以外的说明。
12. 移除全局菜单：注册时可创建标记，移除后右键无响应，重新注册后恢复。
13. 模块菜单与状态清理：先建立车辆轨迹状态，清理后恢复默认，移除菜单后车辆保留但不再响应右键。

## 验证

扩展文档测试，固定每个示例的核心地图操作和全局/模块作用域，防止退化成纯文字反馈。完成后运行全量测试、类型检查、lint 和 `npm run docs:build`。
