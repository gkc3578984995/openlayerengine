# 文档站主题切换设计

## 目标与范围

为 `website/` 文档站提供持久化的浅色/深色主题切换，并将主题适配要求纳入网站维护规范。本次不实现全站搜索。

## 主题行为

- 首次访问默认使用现有浅色主题。
- 用户通过文档站右上角、GitHub 链接旁的太阳/月亮按钮切换主题。
- 选择保存到 `localStorage`；后续刷新或重新打开页面时沿用该选择。
- 应用挂载前读取已保存的选择，并在 `html` 元素上同步添加或移除 `dark` 类，避免先显示浅色再切换深色的闪烁。
- 切换按钮提供可访问名称和提示文本；其图标与当前可执行的切换动作一致。

## 技术设计

1. 在网站入口加载 Element Plus 的深色 CSS 变量。Element Plus 组件随 `html.dark` 自动使用深色变量。
2. 新增主题状态工具，集中处理：默认值、`localStorage` 的安全读取/写入、`html.dark` 的同步和切换。布局组件只消费主题状态，不直接操作浏览器存储。
3. 在 `DocsLayout.vue` 的头部操作区域接入主题按钮，保持现有导航和 GitHub 链接的排列关系。
4. 将站点自身的颜色全部落在语义化 CSS 自定义变量上；在 `html.dark` 下覆盖背景、表面、正文、弱化文本、边框、阴影和强调色。现有组件继续引用这些变量，以使导航、侧栏、页面正文、卡片、代码块与浮动控件共同适配。
5. 为窄屏头部增加相应布局规则，保证主题按钮不会遮挡或挤压导航内容。

## 维护规范

在 `website/AGENTS.md` 增加“主题适配”要求：修改或新增文档站界面必须使用现有语义化主题变量；新增颜色、阴影或表面样式必须同时定义深色值；交付前同时检查浅色与深色主题。禁止以仅适用于单一主题的硬编码颜色替代主题变量。

## 验证

- 为主题状态工具增加单元测试，覆盖默认浅色、保存后恢复、切换时更新 `html.dark` 和存储值。
- 运行网站构建与现有文档质量检查，确认 TypeScript、样式与打包通过。
- 手动确认桌面和窄屏头部的按钮可用，且浅色/深色下主要文档区域可读。

## 非目标

- 不实现全站搜索、搜索索引或搜索输入框。
- 不根据操作系统主题自动选择主题；首次访问固定为浅色，之后以用户显式选择为准。
