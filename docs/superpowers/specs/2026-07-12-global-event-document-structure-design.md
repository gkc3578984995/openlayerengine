# GlobalEvent 地图事件文档结构设计

## 目标

将 GlobalEvent 的文档从按实现入口聚合，调整为按用户处理的事件类型聚合。读者可以先在概览理解自动监听和清理规则，再进入全局鼠标、模块鼠标或全局键盘页面查阅对应 API 与高级控制方式。

## 范围

- 文档显示名称改为 `GlobalEvent 地图事件`。
- 子页面名称改为 `全局鼠标事件`、`模块鼠标事件`、`全局键盘事件`；现有 URL 保持不变，以免已有链接失效。
- `概览与初始化` 只保留构造器和类型定义，不再维护方法分类或任何 `enable*`、`disable*` API 表格行。
- 概览先显示代码演示，后显示 API。
- 概览在“概述”后增加“重要提示：监听自动管理”，放置一段不可运行的极简代码，说明 `add*` 自动启用监听、返回的注销函数只清理本次注册，以及 `disable*` 是会清空同类回调的高级批量操作。
- 原“高级：手动监听控制”独立运行示例和其示例组件删除。
- 每个事件子页面在 API 中同时维护日常 API 与该事件类型的 `enable*`、`disable*` 方法；高级说明与手动监听控制示例放在对应事件页，不放在概览。
- Earth 实例页中 `useGlobalEvent` 的方法名与返回类型跳转到 GlobalEvent 概览的稳定构造器锚点；维护规则同步说明工具页没有方法表时应跳转至其规范入口锚点。
- 更新导航、布局标题、交叉链接、文档维护规则及文档断言测试。

## 信息架构

### 导航

```text
GlobalEvent 地图事件
├─ 概览与初始化
├─ 全局鼠标事件
├─ 模块鼠标事件
└─ 全局键盘事件
```

### 概览与初始化右侧锚点

1. 概述
2. 重要提示：监听自动管理
3. 代码演示
   - 最小完整生命周期
4. API
   - 构造器
   - 类型定义
     - ModuleEventCallbackParams
     - ModuleEventCallback
     - GlobalEventCallback
5. 注意事项

概览的最小示例继续演示日常路径：`earth.useGlobalEvent()`、`add*`、保存返回的注销函数、组件卸载时调用注销函数。它不调用 `enable*` 或 `disable*`。

### API 归属

| 页面 | 常规 API | 高级 API |
| --- | --- | --- |
| 全局鼠标事件 | 所有全局鼠标 `add*` 与 `has*` 方法 | 所有全局鼠标 `enable*`、`disable*` 方法 |
| 模块鼠标事件 | 所有模块鼠标 `add*`、`has*`、`remove*` 方法 | 所有模块鼠标 `enable*`、`disable*` 方法 |
| 全局键盘事件 | `addKeyDownEventByGlobal`、`hasGlobalKeyDownEvent` | `enableGlobalKeyDownEvent`、`disableGlobalKeyDownEvent` |

每个方法只定义一次。子页面的 API 以“日常注册与状态”和“高级：底层监听控制”分组；高级段明确说明 `disable*` 会停止底层监听并清空该类别全部回调，不等价于某一 `add*` 返回的注销函数或模块的 `remove*` 方法。

## 示例策略

- 保留概览的“最小完整生命周期”运行示例。
- 保留现有全局鼠标的持续事件与一次性事件示例，模块鼠标的生命周期与清理范围示例，以及键盘生命周期示例。
- 删除概览的“高级：手动监听控制”运行示例组件。
- 在三个事件子页面的高级说明旁添加对应事件类型的简短手动控制代码块。代码块不是 `ExampleBlock`，因此不进入“代码演示”锚点；它只用于解释 API 的影响范围，不替代日常可运行示例。

## 链接与维护规则

- `useGlobalEvent` 的文字链接和返回类型链接从已删除的 `#api-methods` 改为 `/components/global-event#api-constructor`。
- 其他工具页仍以其实际存在的 `#api-methods` 为链接目标。
- 维护规则改为：工具方法和返回类型必须链接到工具页面的规范入口锚点；若工具有方法表，使用 `#api-methods`，若概览只提供构造器与类型定义，使用 `#api-constructor`。
- 所有示例说明中的方法链接必须指向该方法的唯一归属页；方法表、导航、路由和布局标题必须共同验证。

## 验收与验证

- 文档测试覆盖新显示名称、保留的路由、概览锚点顺序、每个公开 GlobalEvent 方法的唯一表格归属、已删除的独立手动控制示例，以及 `useGlobalEvent` 的新深链接。
- 运行受影响的 Vitest 文档测试，再运行 `npm run verify`、`npm run docs:build` 与 `git diff --check`。
- 完成后播放 Windows 语音通知；不调整系统音量。
