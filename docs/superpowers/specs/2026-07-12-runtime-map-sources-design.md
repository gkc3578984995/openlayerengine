# 文档站运行时地图源配置设计

## 目标

文档站的所有可运行地图示例在打包后仍可切换底图服务，以便部署到内网时替换不可访问的公网瓦片地址，无需重新构建站点。

## 配置文件

新增 `website/public/map-sources.json`。Vite 会原样复制该文件到文档站构建产物根目录；部署人员可直接编辑部署后的 `/map-sources.json`。

配置包含两个具名图层：

- `vector.urlTemplate`：矢量底图 XYZ URL 模板。
- `satellite.urlTemplate`：卫星底图 XYZ URL 模板。
- `satellite.opacity`：卫星层透明度；缺省时为 `0.65`。

URL 模板仅支持 `{z}`、`{x}` 与 `{y}` 占位符。默认配置使用当前可访问的 OSM 与 ArcGIS World Imagery 地址；默认文件不包含 token、账号或内网地址。

## 运行时加载与降级

文档站启动时，在 Vue 应用挂载前请求 `${BASE_URL}map-sources.json`。配置解析模块验证对象结构、URL 模板和透明度范围。

请求失败、JSON 无法解析或字段不合法时，模块使用内置默认配置并输出一次可诊断的警告。应用仍正常挂载，示例不因配置错误而失效。

## 集中底图工厂

新增文档站内部地图源模块，负责：

- 保存默认配置与已加载的运行时配置。
- 将 XYZ URL 模板转换为 `createXyzLayer` 所需的瓦片函数。
- 创建配置化的矢量底图和卫星底图；卫星层应用配置透明度。

所有 `website/src/examples` 中创建底图的示例都必须使用该模块，不得直接写入瓦片服务 URL。此变更仅影响文档站示例，不扩大 Earth 的公开 API。

“管理多个底图”示例继续保存 `addLayer` 返回的句柄：初始矢量层由配置化矢量工厂创建，卫星层由配置化卫星工厂创建。初始“移除矢量底图”按钮可用；“移除卫星底图”仅在卫星层添加后可用。

## 文档与维护规则

“地图创建与销毁”页面新增部署说明，解释构建后编辑 `/map-sources.json` 的方式、占位符规则和错误降级行为。

`website/AGENTS.md` 增加规则：新增或修改带底图的文档示例时，必须通过集中地图源模块读取运行时配置；不得硬编码服务地址或写入私有 token；改动地图源行为时同步更新配置示例与该页面说明。

## 验证

新增单元测试覆盖：

- 有效运行时配置覆盖默认 URL 模板与卫星透明度。
- 缺失或无效配置回退默认值。
- `{z}`、`{x}`、`{y}` 模板替换结果正确。

再运行文档构建，并在本地文档页验证：初始矢量层可移除；添加卫星层后可分别移除两个图层。
