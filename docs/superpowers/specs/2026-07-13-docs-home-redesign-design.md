# 文档首页重设计

## 目标

将当前通用模板式首页改造成能够准确表达 `@vrsim/earth-engine-ol` 定位的精致产品首页，让首次访问者快速理解它是一个基于 OpenLayers 的 TypeScript 地图能力库，并能直接进入安装、地图创建和核心组件文档。

## 设计方向

采用“地图工作台”视觉语言。首屏使用非对称双栏结构：左侧放置品牌叙事、核心行动按钮与技术标签，右侧用纯 HTML/CSS/SVG 构造抽象地图画布、图层面板和坐标状态，避免外部图片依赖并强化地图产品属性。下方用三项能力指标与四张核心模块卡片建立清晰的信息层级。

## 页面结构

1. Hero：小型产品标识、强调 TypeScript 与 OpenLayers 的主标题、简短说明、快速开始与浏览组件两个入口。
2. 地图预览：道路、水域、位置标记、绘制区域、图层列表和坐标信息组成的装饰性工作台；装饰元素对辅助技术隐藏。
3. 能力概览：展示类型安全、模块化能力与真实示例三个优势。
4. 核心模块：链接至地图创建、点图层、测量工具和动态绘制的真实文档路由。

## 视觉与交互

- 使用站点语义主题变量，并为新增表面、网格、光晕和地图元素同时定义浅色与深色值。
- 主色延续现有蓝色体系，加入克制的青绿色作为地图状态色；不引入图片、字体或新依赖。
- 卡片 hover 仅使用轻微位移、边框和阴影变化；尊重 `prefers-reduced-motion`。
- 1180px 以下降低地图预览复杂度，860px 以下改为单栏，560px 以下按钮与信息卡片纵向排列。

## 验收标准

- 所有首页入口指向现有有效路由，不保留赞助商、设计资源或占位入口。
- 浅色与深色主题均具备足够对比度，无单主题硬编码表面。
- 桌面、平板和 320px 起的移动端均无横向溢出。
- 首页结构测试、现有测试与 `npm run docs:build` 通过。
