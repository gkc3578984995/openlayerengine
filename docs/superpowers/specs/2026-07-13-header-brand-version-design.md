# OL-DOC 页头版本标识设计

## 目标

将文档站点左上角品牌文字从 `ol-doc` 改为大写 `OL-DOC`，并在其后显示与根包版本自动同步的版本徽标。当前根包版本为 `1.0.3`，页面显示为 `v1.0.3`。

## 实现设计

- 在 `website/vite.config.ts` 中读取仓库根目录 `package.json` 的 `version` 字段，并通过 Vite 构建常量注入文档站点。
- 在 `website/src/layouts/DocsLayout.vue` 中保留品牌首页链接，将品牌名称改为 `OL-DOC`，并添加独立的版本徽标元素。
- 在 `website/src/assets/styles/index.scss` 中使用已有语义化主题变量设计紧凑的圆角徽标，使其在浅色、深色主题及窄屏下均保持清晰、协调。

## 边界与异常处理

- 版本号只来自根包配置，不在 Vue 组件中重复维护。
- 构建配置读取不到有效版本时应立即报错，避免生成带错误版本标识的站点。
- 本次仅修改左上角页头标识，不调整首页标题、页脚文字或包名。

## 验证

- 运行 `npm run docs:build`，验证 TypeScript、文档同步、API 覆盖和站点生产构建。
- 检查生成内容包含 `OL-DOC` 与当前根包版本 `v1.0.3`。
- 检查徽标样式使用主题变量，并确认现有窄屏规则下不会挤压导航。
