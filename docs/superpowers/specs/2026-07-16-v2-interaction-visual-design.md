# 2.0 Draw / Edit / Transform 交互视觉设计规范

## 文档状态

- 状态：已批准
- 日期：2026-07-16
- 目标版本：@vrsim/earth-engine-ol 2.0.0
- 性质：内部实现规范，不属于 website 用户文档
- 适用范围：Draw、Edit、Transform 三类地图交互及其 OpenLayers、DOM 适配器

## 1. 目标

本规范统一 Draw、Edit、Transform 的视觉反馈和职责边界，解决以下问题：

- 用户进入一种交互后，能够仅凭光标识别当前模式、可操作目标和按下状态。
- Draw 的业务图形预览与最终提交结果保持几何、样式一致。
- Edit 的控制点与插入点具有稳定、可辨认且与设备像素比无关的视觉规格。
- Tooltip 继承既有操作提示，并始终跟随当前指针和会话状态。
- 高频拖拽不会因重复计算范围、重建图层或渲染全部控制点产生明显掉帧。
- 业务拓扑、交互命中、事务提交和 DOM 资源分别由正确的架构层负责。

除非另有说明，本文中的“必须”“不得”和“应”均为实现约束。

## 2. 总体原则

1. **语义一致**：相同光标、视觉和文案在三种交互中必须表达相同含义。
2. **业务预览即结果预览**：预览必须使用元素真实的 `ShapeDefinition` 和解析后样式，不得绘制简化替身。
3. **临时视觉不入库**：预览图形、选中框、控制点、插入点和 Tooltip 都是会话资源，不得写入持久 Store。
4. **拓扑由 Core 决定**：可移动点、可插入位置、可删除点及最小拓扑约束只由 `ShapeDefinition.editTopology` 决定。
5. **显示尺寸使用 CSS 像素**：控制点半径、线宽、Tooltip 偏移和命中容差以 CSS 像素定义，Canvas 绘制时再按 DPR 缩放。
6. **结束必须还原**：交互完成、取消、销毁、被替换或打开失败时，必须清理会话资源并恢复进入交互前的外部光标。

## 3. 统一交互状态

三种交互共享以下抽象状态：

| 状态 | 含义 | 视觉要求 |
| --- | --- | --- |
| `idle` | 当前交互未接管指针 | 使用外部光标，不显示该交互的 Tooltip 或临时图层 |
| `ready` | 会话已打开，尚未命中局部操作点 | 使用模式光标，Tooltip 显示基础操作说明 |
| `hover` | 命中一个可操作点 | 使用该操作的悬停光标，Tooltip 显示局部操作说明 |
| `active` | 主指针已按下并正在操作 | 使用按下光标，Tooltip 显示进行中说明 |
| `ending` | 正在完成、取消或销毁 | 停止接收新输入，释放所有视觉资源，恢复外部光标 |

状态切换必须由会话生命周期驱动。不得依赖一次性的 DOM `mouseover` 或在业务组件外自行推断会话是否结束。

## 4. 光标规范

### 4.1 光标含义

| 光标 | 统一含义 |
| --- | --- |
| 外部光标 | 当前模式没有可操作目标，或交互已经结束 |
| `pointer` | 已进入点击式交互，单击将产生有效操作 |
| `move` | 当前点可拖拽移动 |
| `grab` | 当前旋转手柄可被抓取 |
| `grabbing` | 主指针已按下，拖拽操作进行中 |
| `ew-resize` / `ns-resize` | 水平 / 垂直拉伸 |
| `nesw-resize` / `nwse-resize` | 对应对角方向缩放 |

### 4.2 Draw

- `ready`：`pointer`。
- 绘制过程中继续保持 `pointer`，自由绘制是否按下 Shift 不改变光标。
- 完成、右击退出、取消、销毁、被其他交互替换或打开回滚后：恢复外部光标。

Draw 不使用 `crosshair`。本系统中 `pointer` 明确表达“当前地图点击会增加绘制输入点”。

### 4.3 Edit

| 场景 | 光标 |
| --- | --- |
| 未命中控制点或插入点 | 外部光标 |
| 悬停控制点 | `move` |
| 悬停插入点 | `move` |
| 按下控制点并开始拖拽 | `grabbing` |
| 拖拽结束且仍命中控制点 | `move` |
| 拖拽取消、插入或删除后无命中 | 外部光标 |

只有控制点可以进入拖拽 `active` 状态。插入点的 `move` 表示当前命中了编辑锚点，实际插入仍要求 Alt + 单击；普通单击插入点不得修改拓扑。

### 4.4 Transform

| 操作 | 悬停 | 按下 / 操作中 |
| --- | --- | --- |
| 平移 `translate` | `move` | `grabbing` |
| 顶点编辑 `vertex` | `move` | `grabbing` |
| 旋转 `rotate` | `grab` | `grabbing` |
| 水平拉伸 `stretch-x` | `ew-resize` | `ew-resize` |
| 垂直拉伸 `stretch-y` | `ns-resize` | `ns-resize` |
| 西南—东北缩放 | `nesw-resize` | `nesw-resize` |
| 西北—东南缩放 | `nwse-resize` | `nwse-resize` |

离开手柄后恢复当前会话的外部光标；取消选中、退出 Transform 或会话销毁后恢复进入会话前的外部光标。

### 4.5 光标所有权与恢复

- 会话通过 `CursorPort` 获取独占的 viewport 光标句柄，Session 不直接修改 DOM。
- 公开调用方通过 `ViewService` 更新外部光标；`ViewService` 与 `CursorPort` 必须共享同一基准状态，不能各自直接覆盖 DOM。
- 一个句柄只覆盖自己持有期间的光标。`reset()` 恢复当前外部基准值，`destroy()` 释放所有权。
- 外部组件在交互期间更新 viewport 光标时，适配器必须记住最新外部值；会话结束后恢复该值，而不是硬编码 `auto` 或初始快照。
- 若外部组件已经覆盖当前会话光标，句柄销毁不得再次覆盖外部的新值。
- 所有正常和异常退出路径必须幂等释放句柄。

## 5. 业务图形预览

### 5.1 通用要求

- 预览使用独立的临时渲染层，位于目标业务层之上；不得把工作态 Feature 插入业务 Source。
- 预览几何必须由对应 `ShapeDefinition` 的可信渲染入口生成，适配器不得包含燕尾箭头、进攻箭头等具体图形算法。
- 预览样式必须复用元素解析后的真实样式。线宽、填充、图标、层透明度、可见性、范围和 z-index 语义应与提交后结果一致。
- 每次刷新必须原子替换一份完整工作快照，不得让上一帧几何与下一帧控制点混合显示。
- 会话完成后，只能通过 Session 的 Store 事务出现最终图形；临时预览必须先后安全地移除。
- 没有合法草稿或工作状态时，应清空预览，不得继续显示上一次有效结果。

### 5.2 Draw 预览

- 当前指针只作为预览输入加入控制点序列，只有确认输入才进入 DrawSession 的工作历史。
- 单击、双击、右击结束和自由绘制必须共享同一套几何推导规则。
- 自由绘制或高频移动时，适配器只保留浏览器一帧内的最后一个采样；每个动画帧最多发布一次预览请求。
- `pointerup`、完成或取消前必须同步冲刷最后一个待处理采样，不能因帧合并丢失最终位置。

### 5.3 Edit 预览

- 工作图形使用元素真实样式，持久 Feature 在编辑期间应被隔离或抑制，避免新旧几何重影。
- `underlay` 仅在调用方明确要求时保留原始几何作为参照，不得默认叠加。
- 拖拽中只显示正在移动的控制点；其余控制点和全部插入点暂时隐藏。拖拽结束后恢复完整拓扑锚点。
- 每一帧只更新工作快照，不得提交 Store，也不得为每次移动创建历史事务。

### 5.4 Transform 预览

- 选中框、平移 / 旋转 / 缩放 / 拉伸手柄和复制预览均属于 Transform 会话临时资源。
- Transform 模式的选中框使用半透明灰色填充和灰色描边；活动状态使用虚线描边。Transform Edit 模式不显示选中框，并与独立 Edit 共用第 6 节的蓝色语义锚点。
- 操作中的工作几何只更新临时目标。一次完整操作结束后由 Session 形成一个历史命令；Store 只在整个 Transform 会话完成时提交一次最终事务。
- 跨世界副本只改变展示坐标；业务状态、事务和历史必须保留规范世界坐标。

### 5.5 高频拖拽性能约束

对燕尾箭头等需要根据控制点动态计算几何及范围的元素，必须满足：

1. 指针高频输入按 `requestAnimationFrame` 合并，一帧只消费最后一个位置。
2. 同一帧内只执行一次业务几何推导、一次范围推导和一次渲染提交，不得由命中、Tooltip、控制点图层分别重复推导。
3. 复用临时图层、Feature、Style 和批量控制点渲染器；不得在 `pointermove` 中重建图层或逐点创建 Style。
4. 拖拽过程中冻结无关锚点，只绘制活动控制点；操作结束后再计算并恢复完整锚点集合。
5. 命中测试使用屏幕像素空间的空间索引和有限候选集，不得每次移动遍历并重绘全部业务几何。
6. 帧合并只降低中间预览频率，不得改变最终业务状态、撤销边界或事件顺序。

## 6. Edit 控制点与插入点

### 6.1 视觉规格

| 类型 | 半径 | 填充 | 描边 | 描边宽度 | 虚线 | z-index |
| --- | ---: | --- | --- | ---: | --- | ---: |
| 控制点 | 5 CSS px | `#ffffff` | `#3388ff` | 2 CSS px | 无 | 1 |
| 插入点 | 4 CSS px | `rgba(255,255,255,0.75)` | `#3388ff` | 1.5 CSS px | `[3, 2]` | 0 |

- 控制点表示 `editTopology.describe(state).handles` 返回的现有拓扑点。
- 插入点表示 `editTopology.describe(state).insertions` 返回的合法插入候选，不是普通辅助中点。
- Canvas 批量绘制必须按 DPR 同时缩放半径、线宽和虚线间隔，保证不同设备上的 CSS 视觉尺寸一致。
- 默认控制点视觉命中半径为 6 CSS px，插入点为 4.75 CSS px，再叠加交互 `hitTolerance`；默认 `hitTolerance` 为 8 CSS px。

### 6.2 编辑语义

- 拖拽控制点：发出移动语义，由拓扑 `move` 计算新的 Shape 状态。
- Alt + 单击插入点：发出插入语义，由拓扑 `insert` 计算新的 Shape 状态。
- Alt + 单击可删除控制点：发出删除语义，由拓扑 `remove` 计算新的 Shape 状态。
- 普通单击插入点不得添加点；Alt + 单击不可删除控制点不得删除点。
- Adapter 不得通过坐标相等、首尾点或控制点数量自行判断是否允许插入、删除。

## 7. 命中优先级

命中必须在屏幕像素空间完成，并遵循以下顺序：

1. 先按输入语义过滤候选：普通主键按下准备拖拽时只考虑控制点；Alt + 单击考虑插入点和可删除控制点。
2. 在有效候选中选择与指针欧氏距离最近的锚点。
3. 距离完全相同时，控制点优先于插入点，与视觉 z-index 保持一致。
4. 同类型、同距离候选按稳定的展示顺序决胜，结果不得随 RBush 返回顺序抖动。

指针移动可对所有锚点执行最近命中。插入点不得遮挡同位置或几乎同位置的可见控制点；拖拽开始必须显式优先寻找可移动控制点。

## 8. 跟随鼠标 Tooltip

### 8.1 统一样式与行为

- 每个活动会话最多持有一个 Tooltip，内容和位置原地更新，不得在 `pointermove` 中反复创建 DOM。
- Tooltip 位于指针右上方，偏移为 `[15, -11]` CSS px。
- Tooltip 使用黑色半透明背景、白色 12 px 文字、4 px 圆角和 `4px 8px` 内边距，文本不换行。
- Tooltip 必须设置 `pointer-events: none`、禁止选中，并取消位置过渡动画；高频更新可使用 `will-change: transform`。
- Tooltip 只说明当前有效能力。`ShapeDefinition` 未提供插入或删除能力时，不得显示对应快捷键。
- Tooltip 文案由 Session 按业务状态生成；DOM Adapter 只负责显示，不得推断历史数量、拓扑能力或操作类型。

### 8.2 Draw 文案

基础行按以下顺序组合：

1. `左击开始绘制，右击退出绘制`
2. 自由绘制可用时：`按住 Shift 拖动可自由绘制`
3. 可撤销时：`Ctrl+Z 撤销 (${undoCount})`
4. 可重做时：`Ctrl+Y 重做 (${redoCount})`

Tooltip 在首次有效指针移动时创建，之后持续跟随指针；完成或退出时销毁。

### 8.3 Edit 文案

| 状态 | 文案 |
| --- | --- |
| 悬停插入点 | `按住 Alt 单击添加点` |
| 悬停可删除控制点 | `拖拽控制点编辑图形`；`按住 Alt 单击删除点` |
| 悬停不可删除控制点 | `拖拽控制点编辑图形` |
| 拖拽中 | `拖拽中…` |

未命中锚点时，基础行按能力组合：

1. `拖拽控制点进行编辑`
2. 同时支持插入和删除：`按住 Alt 单击中点添加点 | 按住 Alt 单击可删除控制点`
3. 仅支持插入：`按住 Alt 单击中点添加点`
4. 仅支持删除：`按住 Alt 单击可删除控制点`
5. `右击退出编辑`
6. 可撤销时：`Ctrl+Z 撤销 (${undoCount})`
7. 可重做时：`Ctrl+Y 重做 (${redoCount})`

### 8.4 Transform 文案

未命中手柄时：

- Transform 模式：`选择控制点进行变换操作`
- 快捷键：`Ctrl+C 复制 | Ctrl+V 粘贴 | Ctrl+X 剪切 | Delete 删除 | Esc 退出`
- Edit 模式：`拖拽控制点编辑图形`；按当前拓扑追加插入 / 删除快捷键；`右键完成 | Esc 退出`
- 存在历史时追加 `Ctrl+Z 撤销 (${undoCount})`、`Ctrl+Y 重做 (${redoCount})`。

悬停手柄时：

| 操作 | 文案 |
| --- | --- |
| 平移 | `鼠标左键按下平移` |
| 旋转 | `鼠标左键按下旋转` |
| 水平拉伸 | `鼠标左键按下水平拉伸` |
| 垂直拉伸 | `鼠标左键按下垂直拉伸` |
| 缩放 | `鼠标左键按下缩放，Shift 键保持比例缩放` |
| 顶点编辑 | `拖拽控制点编辑图形` |

操作中：

- 平移：`平移中…`
- 旋转：`旋转中…当前：${degrees}°`
- 拉伸：`拉伸中…`
- 缩放：`缩放中…`
- 顶点编辑：`编辑中…`

### 8.5 Tooltip 生命周期

Tooltip 必须在下列任一情况发生时立即销毁：

- 会话完成或右击退出；
- 会话取消或按 Esc 退出；
- 会话 `destroy()`；
- 交互协调器以新会话替换当前会话；
- 外部目标变更导致当前工作态失效；
- 会话打开过程中任一步骤失败并回滚。

销毁操作必须幂等。Tooltip 不得在会话结束后残留，也不得拦截地图输入。

## 9. 架构边界

### 9.1 ShapeDefinition：拓扑与几何真相

`ShapeDefinition` 是业务规则的唯一真相：

- `editTopology.describe(state)` 返回控制点、`removable`、角色和插入候选。
- `editTopology.move(state, request)` 负责移动控制点。
- 可选的 `editTopology.insert(state, request)` 负责插入点。
- 可选的 `editTopology.remove(state, request)` 负责删除点。
- `toRenderGeometry` 或可信的等价入口负责从 Shape 状态生成业务渲染几何。
- `ShapeRegistry` 必须校验 `vertexEdit`、`controlPointInsert`、`controlPointRemove` capability 与拓扑方法一致。

任何 Adapter 或 Session 都不得复制最小点数、闭合规则、中点构造、燕尾范围算法等 Shape 专属知识。

### 9.2 Adapter：渲染、命中与语义请求

Adapter 只负责：

- 把 Session 提供的展示快照映射到 OpenLayers 临时图层；
- 在屏幕像素空间渲染和命中控制点、插入点及 Transform 手柄；
- 收集浏览器输入，进行动画帧合并；
- 发出冻结的语义请求，如 `pointer-move`、`move-start`、`move`、`move-end`、`move-cancel`、`insert`、`remove`；
- 在结束输入前冲刷最后一个待处理采样。

Adapter 不得直接修改业务 Shape、持久 Feature、Store、历史栈或事务，也不得根据元素类型决定拓扑操作。

### 9.3 Session：工作态、事务与历史

Session 负责：

- 持有当前工作 Shape 和会话状态机；
- 调用 `ShapeDefinition` 校验并执行语义请求；
- 生成一致的图形、锚点、Tooltip 和光标状态；
- 管理撤销 / 重做快照以及事件顺序；
- 在一次操作结束时形成一个历史命令；
- 在会话完成时通过 Store 事务提交一次最终状态；
- 在取消、外部冲突或异常时回滚工作态并清理资源。

拖拽每一帧不是事务，也不是独立历史命令。Adapter 发出的中间采样只能更新 Session 工作态。

### 9.4 DOM 端口与交互协调器

- `CursorPort` 和 Tooltip Port 是唯一允许接触对应 DOM 资源的边界。
- `InteractionCoordinator` 保证 Draw、Edit、Transform 互斥；新会话开始前必须先取消并清理旧会话。
- 所有 Port 句柄都必须支持幂等销毁；Session 的清理顺序不得依赖某个 DOM 节点仍存在。
- 会话结束时恢复最新外部光标，而不是假设 viewport 默认光标为 `auto`。

## 10. 验收清单

实现和回归测试至少覆盖：

- Draw 打开后使用 `pointer`，所有退出路径都恢复进入前或交互期间更新的外部光标。
- Edit 悬停控制点为 `move`、按下拖拽为 `grabbing`、结束后回到 `move`，离开后恢复外部光标。
- Alt + 单击合法插入点能插入；普通单击、不可插入拓扑或控制点命中冲突时不误插入。
- Alt + 单击仅能删除 `removable` 控制点。
- 控制点与插入点重叠时命中稳定，控制点不会被插入点遮挡。
- Tooltip 文案、位置、历史数量和能力提示随状态更新，并在完成、取消、销毁、替换、打开回滚后消失。
- Draw 预览、Edit 工作预览与最终提交结果的几何和样式一致，临时 Feature 从不进入业务 Source。
- 高频拖拽一帧最多触发一次预览更新，`pointerup` 保留最后位置；燕尾箭头等动态范围元素拖拽时不重建图层和全量锚点。
- Transform 各方向手柄光标、进行中文案和结束恢复符合本文表格。
- 外部在会话期间修改 viewport 光标时，结束后保留该外部修改。

本规范只约束内部实现与测试。修复交互缺陷时无需因此更新 `website/`；若未来修改公开 API 或面向用户的操作协议，再单独按仓库文档规范评估 website 同步。
